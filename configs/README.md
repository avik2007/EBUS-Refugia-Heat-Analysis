# configs/ — ArgoEBUS Run Configuration Reference

Each YAML file in this directory tree defines one complete pipeline run.
The filename matches the `run_id` used by `AEResults/aelogs/` exactly, making
config ↔ result pairing unambiguous.

---

## Directory layout

```
configs/
  <region>/
    <region>_<date_start>_<date_end>_res<lat>x<lon>_t<time>_d<d0>_<d1>[<suffix>].yaml
```

Regions match the keys in `ebus_core/ae_utils.py` (`california`, `californiav2`, etc.).

---

## Schema (schema_version: 1)

Both config kinds share the same top-level structure. Unknown fields are rejected.

### Top-level fields

| Field | Type | Description |
|-------|------|-------------|
| `schema_version` | int | Always `1`. Bump + document migration here before changing fields. |
| `config_kind` | `"analysis"` \| `"ingestion"` | Determines which runner is dispatched. |
| `legacy_backfill` | bool | `true` = this config was reconstructed from a pre-manifest run. Skips GPR completeness checks on null fields. |
| `region` | str | EBUS region key. Must exist in `ae_utils.get_ebus_registry()`. |
| `date_start` | YYYY-MM-DD | Inclusive start of the data window. |
| `date_end` | YYYY-MM-DD | Inclusive end of the data window. |
| `lat_step` | float | Spatial grid resolution (degrees latitude). |
| `lon_step` | float | Spatial grid resolution (degrees longitude). |
| `time_step` | float | Temporal bin width (days). |
| `depth_range` | [int, int] | Depth layer bounds in metres, e.g. `[0, 100]`. |
| `run_suffix` | str | Appended to the run_id, e.g. `_3dmatern_w45`. Empty string = canonical run. |
| `description` | str | Free-form annotation. Excluded from the config hash. |

### `input` block (analysis configs)

| Field | Default | Description |
|-------|---------|-------------|
| `source` | `"s3"` | Data source. Only `"s3"` supported now. |
| `s3_path` | — | Full S3 URI to the input parquet. |

### `gpr` block (analysis configs)

| Field | Default | Description |
|-------|---------|-------------|
| `mode` | `"3D"` | GP mode: `"2D"` (spatial only) or `"3D"` (spatio-temporal). |
| `kernel_type` | `"matern0.5"` | Kernel: `"matern0.5"` or `"rbf"`. |
| `window_size_days` | 45 | Rolling window width for temporal GP axis. |
| `step_size_days` | 30 | Step between window centres. |
| `min_bins` | 10 | Minimum occupied spatial bins to attempt kriging. |
| `noise_val` | null | Fixed GP noise variance. `null` = optimise. |
| `time_ls_bounds_days` | null | `[min, max]` bounds on time length-scale. `null` = unconstrained. |
| `lat_ls_bounds` | null | `[min, max]` bounds on latitude length-scale. `null` = unconstrained. |
| `lon_ls_bounds` | null | `[min, max]` bounds on longitude length-scale. `null` = unconstrained. |
| `noise_vals_audit` | [] | Per-window noise values recorded from a completed run. Informational only. |

## Backfill conventions

Configs in this directory that carry `legacy_backfill: true` were generated by
`10_ae_backfill_configs.py` from existing `AEResults/aelogs/` run directories.
Pre-manifest runs did not record GPR hyperparameters, so:

- `noise_val`, `time_ls_bounds_days`, `lat_ls_bounds`, `lon_ls_bounds` are written as `null`.
- `noise_vals_audit` contains the per-window noise values recovered from the audit CSV (what
  the optimizer actually chose each window), which is the closest available evidence.
- The `description` field records the source run_id and the backfill date.

**Do not back-fill these null fields with script defaults.** Writing defaults would misrepresent
what was actually used and corrupt the collision detector's hash-based identity check.

## Schema versioning

`schema_version` is included in the config hash. Any field addition, removal, or semantic change
requires:

1. Bump `schema_version` to the next integer in `config_schema.py`.
2. Add a dated migration note below.
3. Update this README.

## Running a config

```bash
# Validate (no side effects)
conda run -n ebus-cloud-env python ArgoEBUSCloud/aebus_cli.py validate <config.yaml>

# Dispatch analysis end-to-end
conda run -n ebus-cloud-env python ArgoEBUSCloud/aebus_cli.py analyze <config.yaml>

# List registered runs
conda run -n ebus-cloud-env python ArgoEBUSCloud/aebus_cli.py list --region california

# Show a run's manifest
conda run -n ebus-cloud-env python ArgoEBUSCloud/aebus_cli.py show <run_id>
```
